# Task Template

## Task ID
`TASK-XXX`

## Title
Short, action-oriented title.

## Goal
Describe the business/technical goal in 1-3 paragraphs.

## Context
Relevant background the agent should know before implementing.

## Requirements
List the mandatory implementation requirements.

## Inputs
Describe request inputs, filters, params, headers, or dependencies.

## Outputs
Describe the expected output shape, response contract, or resulting behavior.

## API / External Dependency
Describe the external API endpoint(s), auth method, and important integration notes.

## Response Definition
Provide the exact external response (if applicable) and define the mapping contract (DTOs).

## Rules
Important implementation constraints, validations, or architectural rules.

## Acceptance Criteria
- Clear verifiable criteria
- Prefer small bullet points
- Cover behavior, not implementation trivia

## Out of Scope
List what must not be implemented as part of this task.

## Notes
Optional implementation notes, assumptions, or examples.
